# Examen Contra Reloj
## Sistema de Gestión de Paquetes y Sensores en PHP

## Objetivo
El Objetivo es medir la precisión técnica en la sintaxis de PHP y la capacidad de modelado de datos sin usar comportamientos (métodos).

## Problema que resuelve
El proyecto permite organizar y manejar información relacionada con paquetes de envío y sensores de monitoreo de la empresa FastDelivery de forma estructurada, facilitando el acceso y visualización de datos mediante programación Orientada a Objetos.

## Tecnologías Utilizadas
- PHP
- XAMPP
- Apache
- POO

## Conceptos aplicados
- Clases y objetos
- Propiedades públicas y privadas
- Encapsulamiento
- Instanciación de objetos
- Uso de requiere_once
- Manejo de fechas con DateTime
- Strict_types

## Instrucciones de ejecución 

1. Copiar carpeta en htdocts
2. Iniciar Apache en XAMPP
3. Abir localhost/127.0.0.1

   1. Entrar al navegador
   2. Escribir 127.0.0.1
   3. Buscar la carpeta del proyecto
   4. Ejecutar index.php
4. Verificar resultados

## Resultados obtenidos 
Al ejecutar el programa correctamente, se mostraron los datos de almacenamiento en los objetos paquete y sensor, incluyendo el código de seguimiento, peso del paquete, estado de fragilidad, ID del sensor, marca, fecha de última lectura y rango máximo.

## Reflexión personal

### ¿Qué apredí?
La definición de una clase sus atributos y métodos, público, privado, la creación de objetos.
### ¿Qué fue difícil?
Cómo se relacionan varios archivos entre sí usando requiere_once, además de entender la diferencia entre propiedades públicas y privadas dentro de las clases.
### ¿Qué mejoraría?
La presentación visual del programa para que los resultados se muestren de forma ordenada y facl de leer en el navegador.
